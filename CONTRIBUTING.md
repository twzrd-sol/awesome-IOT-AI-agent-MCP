# Contributing to Awesome IoT AI Agents & MCP

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing to this repository.

## 🌟 How to Contribute

### 1. Adding New Resources

We welcome additions in the following categories:

- 🤖 AI Agent Frameworks
- 🔌 MCP Implementations
- 🌐 IoT Platforms
- 📡 Communication Protocols
- 🧠 Edge AI Tools
- 🔒 Security Solutions
- 📄 Research Papers
- 💡 Code Examples

### 2. Contribution Process

1. **Fork the Repository**
   ```bash
   git clone https://github.com/umitkacar/IOT---AI-agent--MCP.git
   cd IOT---AI-agent--MCP
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/add-awesome-resource
   ```

3. **Make Your Changes**
   - Follow the format guidelines below
   - Ensure all links work
   - Add descriptions for new resources

4. **Commit Your Changes**
   ```bash
   git add .
   git commit -m "Add [Resource Name] to [Category]"
   ```

5. **Push and Create PR**
   ```bash
   git push origin feature/add-awesome-resource
   ```
   Then create a Pull Request on GitHub.

## 📝 Format Guidelines

### Adding a Framework/Tool

```markdown
### Framework Name

**Repository**: [owner/repo](https://github.com/owner/repo)
**Documentation**: [Link](https://docs.example.com)
**Stars**: XX.XK+ ⭐

**Key Features**:
- Feature 1
- Feature 2
- Feature 3

**Use Cases**:
- Use case 1
- Use case 2

**Quick Example**:
\```language
// code example
\```
```

### Adding a Research Paper

```markdown
### Paper Title

**Published**: Month Year
**Authors**: First Author et al.
**Venue**: Conference/Journal Name
**Link**: [PDF/ArXiv](https://link-to-paper)

**Abstract**: Brief description of the paper's contribution

**Key Findings**:
- Finding 1
- Finding 2
```

### Adding Code Examples

1. Create file in `/examples` directory
2. Follow existing naming convention
3. Include comments and documentation
4. Add requirements at the top
5. Provide usage instructions

## ✅ Quality Standards

### Before Submitting

- [ ] All links are working
- [ ] Descriptions are clear and concise
- [ ] Proper formatting is used
- [ ] No spelling/grammar errors
- [ ] Code examples are tested
- [ ] New resources are placed in correct category
- [ ] Star counts are accurate (if included)

### Code Quality

- Clear variable names
- Comments for complex logic
- Error handling included
- Compatible with Python 3.8+ or Node 18+
- Dependencies clearly stated

## 🚫 What NOT to Include

- Unmaintained projects (>2 years no updates)
- Commercial-only solutions without free tier
- Duplicate entries
- Self-promotional content without value
- Broken or dead links
- Proprietary/closed-source without open APIs

## 🏆 Recognition

Contributors will be:
- Listed in the repository
- Mentioned in release notes
- Credited in commit history

## 📧 Questions?

- Open an issue for discussions
- Tag maintainers for urgent matters
- Use clear, descriptive titles

## 📜 Code of Conduct

- Be respectful and professional
- Provide constructive feedback
- Welcome newcomers
- Focus on the content, not the contributor

## 🎯 Priority Areas

We especially welcome contributions in:

1. **Real-world case studies** with ROI data
2. **Production-ready code examples**
3. **Performance benchmarks**
4. **Security best practices**
5. **Edge deployment guides**
6. **Recent research papers** (2024-2025)

## 🔄 Review Process

1. Automated checks run on PR
2. Maintainer review (usually within 3 days)
3. Feedback provided if changes needed
4. Merge after approval
